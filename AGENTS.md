# Video Pipeline Agent Workflow

本项目使用“先共创分镜、后实现渲染”的工作流。

当用户提供一条母版口播视频时：

1. 先读取 `docs/镜头语言使用手册.md` 和 `docs/母版分镜共创模板.md`。
2. 转录或读取已有 transcript，按口播语义拆成 beat。
3. 先返回分镜建议表、手机友好的精简审阅卡和素材清单，不直接开始堆视觉效果。
4. 每一镜必须明确：口播锚点、表达任务、唯一主角、`L01–L12` 布局、视觉焦点、展开顺序。
5. AI 创建时补齐六项；用户修改时只需“镜头编号＋改什么”，其余字段继承。
6. 用户以“保留／修改／删特效／并入上一镜／待补”审阅；“删特效”恢复原片，“并入”才延长上一镜。
7. 优先使用 `type: "shot"` 的语义镜头卡；旧组件仍可用于单一特效或兼容旧片。
8. 写完后运行 `npm run validate:shots` 和 `npx tsc --noEmit`；正式渲染前必须通过 `npm run validate:shots:strict`。
9. 用静态镜头目录或 Remotion Studio 的 `ShotLanguageCatalog` 核对布局；用户不必记编号。
10. `L04` 是圆角矩形人物窗，`L05` 才是圆头像。
11. 完整截图只负责建立证据感；需要阅读的区域必须裁切、放大、框选或局部清晰。

所有动画必须由 frame 驱动。保留字幕安全区和人脸保护区；不以颗粒、故障、发光代替镜头结构。
